Panduan Singkat MVC

Dokumen ini menjelaskan cara menjalankan dan menambah halaman pada proyek MVC ini secara ringkas.

1) Jalankan di XAMPP

  • Letakkan folder proyek di: C:\xampp\htdocs\stuarz
  • Buka di browser: http://localhost/stuarz/public/
  • Semua request masuk lewat public/index.php

2) Struktur Folder

app/
  controller/         <- Controller (mis. AuthController.php)
  core/               <- Router.php
  views/
    components/       <- Komponen UI (sidebar, header, dst.)
    layouts/          <- Layout halaman
    pages/            <- File view per halaman (mis. login.php)
public/
  index.php           <- Entry point + Router + Autoloader
  assets/, js/, ...
  panduan/            <- Folder panduan ini

3) Routing

Routing didefinisikan ringkas di public/index.php menggunakan Router:

$router->get('/', 'HomeController@index');
$router->get('/login', 'AuthController@login');
$router->post('/login', 'AuthController@login');
$router->get('/docs', 'DocsController@docs');
// Tambahkan rute lain sesuai kebutuhan...

Format handler: NamaController@metode. Router juga mendukung pola sederhana seperti /news/{id} jika dibutuhkan.

4) Autoloader

Autoloader otomatis memuat class controller tanpa require_once manual. Cukup buat file controller di app/controller/ dengan nama class yang sama (mis. NewsController di NewsController.php).

5) Membuat Halaman Baru

Langkah A: Controller

<?php
class AboutController {
    public function index() {
        include __DIR__ . '/../views/pages/about.php';
    }
}

Langkah B: View

<!-- app/views/pages/about.php -->
<?php $content = __DIR__ . '/about_content.php';
include __DIR__ . '/../layouts/dLayout.php';

Langkah C: Route

// public/index.php
$router->get('/about', 'AboutController@index');

6) Kompatibilitas Link Lama

Link lama dengan ?page=login masih berfungsi. Router memetakan query page menjadi path otomatis.

8) Tips

  • Gunakan kembali layout di app/views/layouts agar konsisten.
  • Komponen UI ada di app/views/components.
  • Tambahkan rute baru di bagian "Define concise routes" pada public/index.php.
